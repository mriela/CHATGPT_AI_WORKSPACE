# Poster Layout Text Draft

## 目的

このドキュメントは、イベント情報確定後にポスター制作へすぐ移れるよう、文字配置とレイアウト構成の初期案を整理するものです。

## 基本レイアウト案 A：中央タイトル型

### 構成

- 上部：イベント種別 / キャッチコピー
- 中央：イベント名を大きく配置
- 下部：開催日時・会場・主催者情報
- 右下または左下：QRコード / 補足情報

### 向いている用途

- イベント名を強く印象づけたい場合
- SNS告知画像
- Unity内掲示ポスター

## 基本レイアウト案 B：ビジュアル主役型

### 構成

- 背景または中央にメインビジュアル
- イベント名は上部または下部に大きく配置
- 日時・会場は読みやすい帯エリアに配置
- 細かい情報は最下部にまとめる

### 向いている用途

- キャラクター画像や背景画像が強い場合
- 雰囲気重視のイベント告知
- VRChat向けの視覚的訴求

## 基本レイアウト案 C：情報整理型

### 構成

- 上部：イベント名
- 中央：日時・会場・出演者をブロック化
- 下部：参加方法・注意事項・QRコード
- 背景は控えめにする

### 向いている用途

- 情報量が多いイベント
- 初参加者向け告知
- 読み間違いを避けたい場合

## 文字要素テンプレート

```text
[キャッチコピー]

[イベント名]

開催日：
開始時刻：
終了時刻：

会場 / ワールド：
主催：
出演 / 参加：

[告知文]

[参加方法 / 注意事項]

[QRコード または URL]
```

## Unity掲示用の注意

* 遠目でも読めるように、イベント名・日時・会場を最優先にする
* 細かい文章は入れすぎない
* QRコードを使う場合は十分な余白を取る
* 文字色と背景色のコントラストを強くする

## SNS用の注意

* 最初に目に入る要素は「イベント名」または「メインビジュアル」にする
* 日時は必ず大きめに配置する
* 詳細情報は投稿本文に逃がしてよい
* 画像内テキストは最小限にする

## 次に必要な作業

1. `poster_event_info_template.md` に実イベント情報を記入する

2. 使用するレイアウト案を A / B / C から選ぶ

3. 必須素材を `assets/poster/references/` に置く

4. 初回デザイン案を `assets/poster/drafts/` に作成する
