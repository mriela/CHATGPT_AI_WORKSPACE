# Poster Asset Folder Guide

Purpose:
- Define the folder structure for managing VRChat music event poster assets on MRIELA.
- Make it clear which files should be passed to SPALITUDA for Unity venue integration.

## drafts/
- Store work-in-progress poster drafts.
- Use this folder for layout tests, rough exports, and design variations.

## references/
- Store reference images, mood boards, color references, and visual notes.
- Keep source URLs or permission notes nearby when needed.

## exports/
- Store final poster exports for announcement use.
- Use clear filenames that include `_final` and the intended size or platform when useful.

## source/
- Store editable source files such as design project files, layered images, and text notes.
- Keep this folder organized so later revisions can be made without rebuilding from scratch.

## unity-ready/
- Store poster textures and sign variants prepared for Unity import.
- Keep files optimized, readable at VRChat viewing distance, and aligned with the selected venue direction.

## Handoff Notes
- SPALITUDA should mainly receive files from `exports/` and `unity-ready/`.
- Include source files from `source/` only when further editing is expected.
- Keep drafts and references available for context, but avoid treating them as final deliverables.
